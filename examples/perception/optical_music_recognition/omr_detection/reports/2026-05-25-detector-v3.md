# Detector v3 — retrained on cleaned GT + 2x data

## Setup

- Data: L7a (1000 train / 100 dev) + L9 (1008 / 100) + openscore (567 / 77) = **2575 train / 277 dev**.
- GT pipeline: lilypond --svg → cairosvg, plus 3 new fixes since v2:
  - `MIN_STAFF_LEN` 50 → 25 mm (catches short final-system staves)
  - Trailing partial systems no longer dropped
  - System bbox includes 1.5 staff-space Y margin for ledger notes
- Same 19-class multitask scheme (system, staff, barline_single, barline_heavy, key_-7..+7).
- yolov8n, 60 epochs, imgsz=1280, batch=16, lr=1e-3.

## What moved the needle

Mostly the keysig sub-classes for rare fifths values (+/-5, +/-6, +/-7). v2
had 0 mAP on key_+7 because only 6 instances were in the dev set and no
training instances. v3 caught more rare-key examples in the bigger train
set + cleaner GT, lifting those classes from broken to ~0.92.

Well-represented classes were already at 0.995 in v2 and stayed there.

Structural classes (system, staff, barline_single) hit 0.995 immediately
and remain perfect.
