# vlm_omr_per_measure_sft: design notes

## Why now

Multi-task detector + safckylj VLM hits ~95% pitched on L7a but **~2%
pitched on openscore** (measured 2026-05-23, 3-page sample). The page-level
VLM doesn't generalise out of its training distribution; respell can't
help because the underlying notes are hallucinated, not just mis-keyed.

The detector workstream now gives us reliable cell bboxes. So we can
crop per-measure, transcribe per-measure, stitch ŌĆö a fundamentally
different training distribution that may be much more tractable for
real-world music.

## Cross-measure handling: per-measure outputs partial endpoints

MXC2's existing tokens make per-measure decomposition clean:

| Element | First measure emits | Next measure emits |
|---|---|---|
| Tie       | `tie=start` on last note | `tie=stop` on first note |
| Slur      | `slur1=start` somewhere  | `slur1=stop` (potentially many measures later) |
| Wedge     | `wedge=crescendo`        | `wedge=stop` |
| Beam      | `bm=begin` / `bm=continue` | usually doesn't cross barlines |
| Repeat    | drawn at barline; per-measure crop sees it | ŌĆö |
| Endings   | `ending=` at barline | ŌĆö |

Stitcher = concat per measure in reading order, per part. No reconcile
pass needed: MXC2 endpoints are positionally independent.

## What MXC2 cannot represent (could matter on real openscore)

| Element | Pitch impact | Frequency in openscore lieder | Plan |
|---|---|---|---|
| **Ottava (8va / 8vb)** | **Yes** -- displayed vs sounding pitch | Common in piano accompaniment | **Phase 1 extension if measured to matter** |
| Pedal markings | No | Common | Skip (no pitch impact) |
| Cross-staff routing | No (pitch unchanged), wrong-staff cell assignment | Common in piano | Defer |
| Glissando | No | Rare | Skip |
| Trill extension | No | Rare | Skip |
| Multi-measure rest | Compresses many measures | Orchestral, rare in lieder | Skip |

Ottava is the one that's both common in our target data and
pitch-affecting. If the per-measure VLM's openscore errors cluster around
notes inside 8va brackets, we extend MXC2 with a per-note `oct=┬▒8` tag
(backward-compatible, opt-in via a `version="2.1"` flag in `xml_to_mxc2`).

## Training-time choices

### Initial base: safckylj (Qwen3-VL-8B)
Continues from a known-good checkpoint. Fastest to validate viability.
Per-measure SFT should converge in O(1) GPU-day on L7a + L9 + openscore.

### Per-measure vs per-cell
- **Per-cell** (staff ├Ś measure): one staff worth of music; clean MXC2
  slice via `mxc2_slice`; loses cross-staff chord context.
- **Per-measure** (whole system ├Ś measure): captures all staves at that
  time-slice; the slicer handles multi-staff voices correctly.

Going with per-measure first -- the cross-staff context matters for
piano chord voicings, and the slicer already handles multi-part output.

### Max output tokens
A measure with ~8 chord notes + ties + slurs is well under 100 tokens.
Cap `max_new_tokens=256` for headroom. Compare to 8192 for whole-page.

### Sample count
- L7a: ~500 pages ├Ś ~16 measures ├Ś 3 staves = ~24K cells (or ~8K
  measures if per-measure crops)
- L9: ~300 pages ├Ś ~16 measures ├Ś 3-4 staves = ~14K cells / ~5K measures
- openscore: ~150 unique pages ├Ś ~10 measures ├Ś 2-3 staves = ~4K cells
  / ~1.5K measures

Per-measure total: ~15K training samples. Plenty for SFT.

## Inference path

```
page image -> multi-task YOLO -> systems, staves, barlines, keysig boxes
                              -> derive_measures()        -> list[Measure]
for each Measure m:
    crop = page.crop(m.bbox)
    measure_mxc2 = VLM(crop, instruction="Transcribe single measure ...")
stitched = stitch_measures(per_measure_outputs)
respelled = respell_mxc2(stitched, key_from_detector)
```

Stitching: concatenate per measure in reading order, restating part
headers (`P1 Voice`, `P2 Piano RH`, ...) at the top and emitting
`---` once. The slicer already produces per-measure slices with
restated state; the stitcher does the inverse.

## Risk

The biggest gamble is whether **per-measure crops carry enough visual
context** for a small VLM trained from scratch to learn the encoding.
Two failure modes to watch for at first SFT epoch:
1. Model can't read note duration / pitch from a tight crop (resolution
   too small). Mitigation: increase imgsz; crop with vertical padding
   for ascender/descender ledger lines.
2. Model loses chord-voicing accuracy when cross-staff chords get
   split. Mitigation: switch to per-cell + add lower-staff visual
   context as a side crop.

We'll know within one training run.
