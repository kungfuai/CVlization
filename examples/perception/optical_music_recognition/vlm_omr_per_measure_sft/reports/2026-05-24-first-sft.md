# First per-measure SFT runs (v1 + v2)

## Setup

SFT'd `safckylj` (Qwen3-VL-8B) on per-measure cells from L7a + L9 +
openscore. Reused safckylj's existing LoRA adapters (no fresh
get_peft_model call needed -- it errors with "You already added LoRA
adapters" on an unmerged final_model).

| run | dataset | epochs | headers | train loss |
|---|---|---|---|---|
| v1 | `/tmp/per_measure_v1` (18k train / 3.6k dev) | 1 | included | 0.296 final |
| v2 | `/tmp/per_measure_v2` (18k / 3.6k, stripped) | 2 | stripped | (intermediate logs lost) |

`v2` strips per-score `header *` lines (work-title, composer, lyricist,
movement-title) from the MXC2 target -- those are not recoverable from
a single-measure crop. On v1 the model hallucinated wrong headers
(e.g., predicted Brahms "Die Schwalbe" for a Bishop "Home Sweet Home"
crop).

## Results (12 dev cells per source)

| source | metric | v1 (1ep, headers) | v2 (2ep, no headers) | safckylj whole-page (ref) |
|---|---|---|---|---|
| L7a       | pitch  | 69.1% | 68.7% | ~95% |
| L7a       | rhythm | 100%  | 100%  | ~100% |
| L9        | pitch  | 30.7% | 26.8% | (~95% on whole-page) |
| L9        | rhythm | 49.3% | 62.2% | |
| openscore | pitch  | 15.7% | 16.9% | **2%** |
| openscore | rhythm | 47.2% | 58.5% | (very low) |

**Wins**:
- Openscore pitch: ~2% (whole-page) -> ~17% (per-measure). 8.5x lift,
  same VLM, smaller compute per inference. Confirms the per-measure
  approach is the right axis for out-of-distribution generalisation.
- L7a/L9 rhythm: 100% / 62% -- the model learned the per-measure
  format and emits sensible MXC2 structure.

**Stalls**:
- L7a pitch: 95% (whole-page) -> 69% (per-measure). The per-measure
  model is *worse* on its own training-distribution data. Per-measure
  crops are ~280x100 px; whole-page is 1240x1754. Pitch errors cluster
  on closely-spaced noteheads where the per-measure resolution is too
  low to disambiguate (e.g., B4 vs C5).
- Openscore pitch: 17% is far from useful.

## Diagnostics

L7a EXACT matches: 3/12 cells (clean voice + piano measures). Failures
are uniform pitch errors with perfect rhythm -- model knows the meter
and stem direction; just mis-identifies the note within a few staff
spaces.

## Next experiments

In rough order of expected payoff:

1. **Pad cell crops with vertical context** -- include partial staves
   above + below the measure cell so the vision tower has more pixels
   to work with. Same musical content, more visual signal.
2. **Upsample crops at inference** -- resize each cell crop up to a
   minimum height (e.g., 384 px short side) before feeding to VLM.
   Probably the cheapest win.
3. **More epochs / larger LR sweep** -- v2 is 2 epochs; training loss
   may not have plateaued. Try 5+ epochs.
4. **Wider openscore training data** -- currently 150 unique scores;
   the dataset has thousands more.
5. **Smaller architecture** -- 8B is overkill for measure-sized inputs.
   A 3B or smaller VLM might train faster and serve faster, freeing
   us to iterate more.
