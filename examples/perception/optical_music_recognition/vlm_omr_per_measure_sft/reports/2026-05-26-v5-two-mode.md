# Per-measure VLM v5 — two-mode prompt + diagnosis

## Setup

Same dataset as v4 (39942 train + 4387 dev per-measure samples from
det_v3 derivatives). Training change: `--two-mode --first-oversample 4`
— first-of-page measures use the part-names-only prompt
(model must infer key/time/clef from the image); rest of measures keep
the full active-header context. 47,667 train samples (10,300 of which
are oversampled first-of-page).

1 epoch, batch 4 × grad_accum 4, lr 2e-5, same LoRA config as v4.

## End-to-end check on openscore lc6211535 p1 (Beach)

|metric|v4|v5|GT context|
|---|---|---|---|
|pitched_only_similarity|0.080|**0.017**|↓ regression|
|key_ok|✓|✓|key 0 (C major)|
|time_ok|**✗** (pred 4/4)|**✓** (pred 2/2)|GT 2/2|
|inferred clef pair|G2,F4 (default)|G2,F4 (still default)|GT G2,G2 (rare!)|
|pred events|288|252|GT 269|
|measure numbers contiguous|yes|**no (skips m10, m16)**|—|

v5 *did* learn to infer time signature from m1's image (huge win — m1
of any score visually shows the time-sig glyphs). But:

1. It still defaults to G2+F4 for the piano clef pair because training
   was dominated by treble+bass piano scores. The Beach piece uses two
   treble clefs (unusual) so the model never saw enough G2+G2 examples
   to learn it.
2. Some measures produce malformed / empty output that stitch_measures
   drops, leading to non-contiguous measure numbers and a shorter
   document. This kills SequenceMatcher alignment even when the per-
   measure content is similar to v4.

## What's needed to push past 0.1

- **Clef-per-staff detector** — extend YOLO with G2/F4/C3/C4/perc
  classes. Feed correct clef into the prompt instead of defaulting to
  G2+F4. This would eliminate the 2-octave shift on second staff.
- **Stitch robustness** — handle malformed per-measure output without
  silently dropping it (insert a placeholder M-line with rests so
  measure numbering stays contiguous).
- **Data balance** — augment with more G2+G2 piano examples (or other
  rare clef pairs). Currently training mostly sees grand-piano with
  treble+bass.

## Checkpoints

- `outputs/per_measure_v5/final_model/` — two-mode v5 (this report)
- `outputs/per_measure_fresh_v4/final_model/` — v4 (always-header,
  needs correct defaults at inference)

Empirically v4 is still slightly stronger on pitched-similarity for
this single test page, because v5's regression from header-default-
correction was outweighed by the stitch-dropping behavior. But the
inference contract is cleaner for v5: no hand-tuned defaults needed.
