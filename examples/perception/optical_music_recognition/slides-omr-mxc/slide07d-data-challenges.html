Dataset Challenges and Scale

Engineering challenges

  • Render failures: 81/1,460 lieder, 4/122 quartets, 2/94 orchestra scores failed LilyPond rendering (syntax errors, zero-duration objects)
  • music21 slicing bugs: arpeggio spanners cause TypeError; retry with gatherSpanners=False. "Measure already in stream" on some page boundaries
  • SVG bar number parsing: time signature denominators have same font size as bar numbers; disambiguate by keeping largest consecutive-integer cluster
  • Pickup bars: 23% of lieder have measure 0; auto-detect and offset SVG bar numbers by -1
  • Runtime: 12-20 hours per corpus (Docker render + music21 slicing)

Final dataset stats

Lieder: 3,312 train / 209 dev / 223 test

Quartets: 6,274 train / 136 dev

Orchestra: 4,870 train / 242 test

Data quality

  • 3/500 lieder samples have malformed XML (music21 export bugs)
  • 7/27,632 lyric elements contain non-breaking spaces
  • 224 quartet pages skipped (page-number-only, no bar content)
  • All issues documented in page_musicxml_runs.md for future fixes
